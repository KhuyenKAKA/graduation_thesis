Chatbot Processing Pipeline

1
User Input & Session Restoration
User submits a message via the chat interface. The backend receives the message, session_id, and user_id, then restores session context: conversation history, history summary, and the user's academic profile.
Vue.js Frontend FastAPI Backend MySQL
2
Intent Recognition
A bilingual keyword classifier analyzes the input to determine whether it is a domain-specific query requiring database retrieval or a general conversational message. Reduces unnecessary SQL calls and improves response speed.
Domain Query General Chat
3
Data Retrieval — Text-to-SQL
For domain queries, the system sends the question with full DB schema, SQL rules, examples, and conversation history to the LLM to generate a SQL statement. The query is security-validated (SELECT-only) before execution against MySQL.
Text-to-SQL Security Validation MySQL
ChromaDB
(RAG)
4
Context Construction
The final prompt is assembled from: SQL query results, supplementary descriptive text retrieved from ChromaDB via cosine similarity search on sentence embeddings, system prompt, conversation history summary, and user academic profile.
RAG (ChromaDB) SQL Results User Profile History Summary
5
Response Generation
The assembled prompt is sent to the LLM in streaming mode. Response tokens are generated and streamed incrementally. An automatic retry mechanism handles temporary API failures.
LLM (Gemini) Streaming Auto-retry
6
SSE Stream Delivery
Each response chunk is immediately forwarded to the frontend via Server-Sent Events, producing a typewriter effect. Special events (online search triggers, session completion signals) are embedded in the stream. FastAPI's async architecture supports concurrent users without blocking.
SSE FastAPI Async Vue.js Frontend
7
Storage & History Summarization
The full conversation turn is persisted to MySQL. When a session accumulates 10 or more new messages since the last summary, the LLM compresses the conversation into a concise 3–5 sentence summary, which is loaded at the start of each subsequent request to maintain long-term context without inflating prompt size.
MySQL LLM Summarization Context Management
